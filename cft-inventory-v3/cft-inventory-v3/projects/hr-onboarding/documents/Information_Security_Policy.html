Information Security Policy
Rules for Handling Company Data, Devices, and Systems
Effective Date: _____________ | Version 1.0
1. Purpose & Scope

This Information Security Policy establishes rules and procedures to protect Craftech360 Private Limited ("Company") information assets, including data, systems, and devices, from unauthorized access, disclosure, modification, or destruction.

This Policy Applies To:
  • All employees, contractors, interns, and temporary staff
  • All Company-owned devices, systems, and networks
  • Personal devices used for Company work (BYOD)
  • All data created, stored, or transmitted in the course of Company business
  • Third-party systems and cloud services used for Company purposes
2. Data Classification

All Company information is classified into the following categories:

Classification Description Examples Handling
CONFIDENTIAL Highly sensitive; unauthorized disclosure could cause significant harm Financial data, salaries, client contracts, source code, trade secrets Encrypted storage; need-to-know access only; no sharing externally
INTERNAL For internal use; not intended for public disclosure Internal communications, project plans, vendor lists, employee contacts Access limited to employees; do not share externally without approval
PUBLIC Approved for public distribution Marketing materials, website content, press releases Can be shared freely; must be accurate and up-to-date

When in doubt, treat information as INTERNAL.

3. Password & Authentication
🔐
Password Requirements
  • Minimum 12 characters for all accounts
  • Mix of uppercase, lowercase, numbers, and special characters
  • No dictionary words, names, or easily guessable information
  • Change passwords every 90 days
  • Never reuse passwords across different systems
  • Use a password manager (Company-approved: Bitwarden)
🚫 Never:
  • Share your password with anyone, including IT staff or managers
  • Write passwords on sticky notes or store them in plain text
  • Use the same password for work and personal accounts
  • Send passwords via email or chat
Multi-Factor Authentication (MFA)

MFA is mandatory for:

  • All email accounts (Google Workspace)
  • Cloud storage and collaboration tools
  • Remote access / VPN
  • Administrative access to any system
  • Financial and HR systems
4. Device Security
4.1 Company-Owned Devices
  • All laptops must have full-disk encryption enabled
  • Screen lock must activate after 5 minutes of inactivity
  • Antivirus software must be installed and kept updated
  • Operating system and applications must be updated within 7 days of security patches
  • Do not disable security features or install unauthorized software
  • Report lost or stolen devices to IT immediately (within 1 hour)
4.2 Mobile Devices (Phones/Tablets)
  • Enable device PIN/biometric lock
  • Enable remote wipe capability
  • Do not jailbreak or root devices used for work
  • Only install apps from official app stores
  • Enable automatic updates
4.3 Personal Devices (BYOD)

If using personal devices for Company work:

  • Must meet same security standards as Company devices
  • Company may remotely wipe Company data if device is lost/stolen
  • Do not store Confidential data on personal devices without encryption
  • Separate work and personal accounts where possible
  • Notify IT before using personal device for work
5. Data Handling & Storage
Activity ✓ Allowed ✗ Prohibited
Cloud Storage Company Google Drive, approved cloud services Personal Dropbox, WeTransfer, unapproved services
File Sharing Google Drive sharing (with proper permissions), Company email Public file sharing links, personal email for work files
USB Drives Encrypted drives for approved purposes only Personal/unknown USB drives; unencrypted transfer of Confidential data
Printing Secure printing for necessary documents Leaving Confidential printouts unattended
Disposal Secure deletion, shredding physical documents Throwing documents in regular trash, simple file deletion
⚠️ Client Data: Client project files, creative assets, and proprietary content are CONFIDENTIAL. Never share with unauthorized parties, post publicly, or include in portfolios without written client consent.
6. Email & Communication Security
  • Use Company email for all work communication
  • Verify sender before clicking links or opening attachments
  • Do not forward work emails to personal accounts
  • Encrypt emails containing Confidential information
  • Be cautious of phishing attempts — report suspicious emails to IT
  • Do not send Confidential data to external recipients without encryption or secure file sharing
Phishing Red Flags:
  • Urgent requests for passwords or financial information
  • Unexpected attachments or links
  • Spelling errors or unusual sender addresses
  • Requests that bypass normal procedures
7. Network & Internet Security
7.1 Office Network
  • Connect only Company-approved devices to the network
  • Do not share WiFi credentials with visitors without authorization
  • Do not attempt to bypass network security controls
7.2 Remote Work & Public WiFi
  • Use VPN when connecting to Company systems remotely
  • Never access Confidential data on public WiFi without VPN
  • Ensure home WiFi uses WPA2/WPA3 encryption with strong password
  • Do not work on sensitive matters in public spaces where screens are visible
7.3 Acceptable Use

Company internet may be used for:

  • Work-related activities
  • Limited personal use that does not interfere with work

Prohibited: Accessing illegal content, downloading pirated software/media, streaming that impacts bandwidth, gambling, or any activity that violates Company policies.

8. Software & Access Management
  • Only use Company-licensed software
  • Do not install unauthorized software or browser extensions
  • Request software through IT; do not download from unofficial sources
  • Access is granted on a need-to-know basis
  • Report changes in job role that affect access needs
  • Do not share login credentials or access tokens
  • Log out of systems when not in use; especially shared devices
9. Physical Security
  • Lock workstation when leaving desk (Win+L / Cmd+Ctrl+Q)
  • Do not leave devices unattended in public places
  • Secure laptops with cable lock when in office (if provided)
  • Do not allow tailgating — challenge unknown individuals
  • Store Confidential documents in locked cabinets
  • Clear desk of sensitive materials at end of day
  • Report suspicious persons or activities to security
10. Incident Reporting

Report security incidents immediately to IT and HR. Examples include:

  • Lost or stolen device
  • Suspected unauthorized access to systems
  • Accidental data disclosure
  • Suspicious emails or phishing attempts
  • Malware or virus infection
  • Physical security breaches

11. Consequences of Violation

Violation of this Policy may result in:

  • Disciplinary action up to and including termination
  • Civil or criminal liability for data breaches
  • Personal liability for damages caused by negligence
  • Reporting to law enforcement where applicable

Severity of consequences will be determined based on the nature of the violation, intent, and impact.
