# Craftech360 Event Project Workflow

*Source: Suman (HR) — 2026-02-07*

---

## Event Lifecycle: Client Inquiry → Delivery

### 1. Client Inquiry & Requirement Understanding

**Entry points:**
- Referral
- Website
- LinkedIn
- Direct contact

**BD/Client Servicing gathers:**
- Event objective
- Event type & scale
- Location, dates, budget
- Technology / creative expectations

---

### 2. Internal Discussion & Feasibility Check

**Shared with:**
- Creative Team
- Tech / Solutions Team
- Operations Team

**Assessed:**
- Feasibility
- Timelines
- Inventory availability
- Technical complexity

---

### 3. Concept, Proposal & Costing

- Creative concept + technical solution created
- Budget, timelines, and scope finalized
- Proposal and deck shared with client

---

### 4. Client Approval & Closure

- Revisions (if any)
- Final approval on:
  - Concept
  - Commercials
  - Deliverables
- Agreement / PO confirmation received

---

### 5. Project Planning & Internal Alignment

- Project kick-off meeting
- Responsibilities assigned:
  - Creative
  - Tech
  - Inventory
  - Operations
  - Client Servicing
- Event checklist and timeline created

---

### 6. Execution & Event Delivery

- Inventory preparation & testing
- Installation and setup on-site
- Live event execution with tech and ops support

---

### 7. Event Closure & Post-Event Activities

- Dismantling & inventory return
- Post-event report
- Client feedback and closure
- Billing and payment follow-up

---

## Key Handoffs

| Stage | From | To |
|-------|------|-----|
| Inquiry | Client | BD/Client Servicing |
| Feasibility | BD | Creative + Tech + Ops |
| Proposal | Creative + Tech | BD → Client |
| Approval | Client | Internal teams |
| Execution | All teams | On-site delivery |
| Closure | Ops | Finance + BD |

---

*This workflow is the backbone of Craftech360's service delivery.*
